# Get Shit Done (GSD) Workflow Guidance

You are operating within a GSD project workspace. Follow these principles:
- **Phase-Driven**: Work is organized into phases defined in `.planning/ROADMAP.md`.
- **Requirements-First**: All work must map to requirements in `.planning/REQUIREMENTS.md`.
- **State Tracking**: Update `.planning/STATE.md` with progress and blockers.
- **Atomic Commits**: Commit work logically and regularly.
- **Workflow Steps**: 
  1. `/gsd-discuss-phase` to clarify the approach.
  2. `/gsd-plan-phase` to create a concrete plan.
  3. Execute the plan.
  4. Transition to the next phase.
